## Quick Start

### Installation

```bash
npm install @adaptive-cards/ui
```

### Usage

```tsx
import { Card, CardHeader, CardTitle, CardContent, Button } from '@adaptive-cards/ui';
import '@adaptive-cards/ui/styles.css';

function MyComponent() {
  return (
    <Card variant="default" size="md">
      <CardHeader>
        <CardTitle>Welcome to Adaptive Cards</CardTitle>
      </CardHeader>
      <CardContent>
        <p>This is a sample adaptive card with modern design.</p>
        <Button className="mt-4">Get Started</Button>
      </CardContent>
    </Card>
  );
}
```

## Design Principles

### 1. Accessibility First
- All components meet WCAG 2.1 AA standards
- Keyboard navigation support
- Screen reader compatibility
- Focus management and indicators
- High contrast mode support

### 2. Mobile-First Responsive
- Touch-friendly interaction targets (44px minimum)
- Responsive typography and spacing
- Flexible layouts that adapt to any screen size
- Performance optimized for mobile devices

### 3. Consistent Visual Language
- Microsoft-inspired design with modern touches
- Consistent spacing based on 8px grid system
- Semantic color system with proper contrast ratios
- Elevation system for visual hierarchy

### 4. Developer Experience
- TypeScript-first development
- Comprehensive documentation with examples
- Consistent API patterns across components
- Easy customization with CSS custom properties

## Component Architecture

### Base Components
- **Card** - Container component with variants and interactive states
- **Button** - Action component with loading states and icon support
- **Input** - Form input with validation and accessibility features
- **Badge** - Status and labeling component
- **Skeleton** - Loading state placeholder

### Composition Patterns
Components are designed to work together seamlessly:

```tsx
<Card interactive onClick={handleCardClick}>
  <CardHeader>
    <div className="flex justify-between items-start">
      <CardTitle>Project Status</CardTitle>
      <Badge variant="success">Active</Badge>
    </div>
  </CardHeader>
  <CardContent>
    <Input 
      label="Project Name"

    />
  </CardContent>

```

## Customization

### CSS Custom Properties
The design system uses CSS custom properties for theming:

```css
:root {
  --primary: 207 100% 42%;      /* Microsoft Blue */
  --secondary: 210 40% 96%;
  --success: 142.1 76.2% 36.3%;
  --warning: 32.1 94.6% 43.7%;
  --error: 0 84.2% 60.2%;
  --radius: 0.5rem;
}
```

### Dark Mode
Dark mode is automatically handled through CSS classes:

```tsx
<div className="dark">
  {/* Components automatically adapt to dark theme */}
  <Card>Dark mode card</Card>
</div>
```

### Extending Components
Use class-variance-authority for consistent component variants:

```tsx
import { cva } from 'class-variance-authority';
import { Card } from '@adaptive-cards/ui';

const customCardVariants = cva('', {
  variants: {
    brand: {
      primary: 'border-brand-500 bg-brand-50',
      secondary: 'border-gray-300 bg-gray-50',
    }
  }
});

function CustomCard({ brand, ...props }) {
  return (
    <Card 
      className={customCardVariants({ brand })}
      {...props}
    />
  );
}
```
